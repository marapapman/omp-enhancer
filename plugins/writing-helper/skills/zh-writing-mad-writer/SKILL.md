# 中文快速写作技能

批量写中文文档章节，默认执行一轮写、查、改。读取叙事和文献上下文，完成目标章节，做 7 维度聚焦检查，能修的直接修，修不了的标记出来。只有用户明确要求迭代修订时才开始下一轮。

## 输入

- paper.md（必须有）— 当前文档
- .pi/research/storyline.md（如果存在）— 研究叙事
- .pi/research/literature.md（如果存在）— 相关工作上下文

## 工作流程

### 第 0 轮：准备

读取所有输入。识别空章节（空白或 `[TODO]`/`[FILL]` 标记）。如果缺少相关工作文献：从 storyline 提取 3-8 个关键词，每个关键词用 `web_search_exa("<关键词> site:arxiv.org", max_results=3)` 搜索，用 `web_fetch_exa` 获取摘要，写入 `.pi/research/literature.md`（每次最多 10 篇）。

### 默认一轮：写 → 查 → 改

**1. 写作冲刺** — 对每个空/稀疏章节：读取上下文 + `<!-- description: ... -->`，写主题句（第 6 级标题，≤50字符）+ 正文。正文长度服从语义完整和章节需要，通常不超过 800 字。引用文献。

**2. 内嵌审查** — 运行全部 7 项检查。每个发现引用具体证据。不捏造问题。

| 维度 | 检查内容 |
|------|----------|
| 问题 | 定义清楚？重要性有证据？关联具体需求？ |
| 新颖性 | 与已有工作区分？指出具体不足？ |
| 深度 | 解决了有难度的技术问题？ |
| 逻辑 | 论证一致？论点有证据？无跳跃或矛盾？ |
| 清晰度 | 术语有定义？语言精确？无翻译腔和 AI 味？ |
| 评估 | 实验对应研究问题？基线合理？指标合适？ |
| 数据 | 引用真实？复现路径清晰？ |

严重程度：CRITICAL（影响整个论点）/ IMPORTANT（需要修复）/ MINOR（改进建议）。

**3. 能修的自动修** — 可修的：术语不清→定义、模糊→具体、缺失→补充、稀疏→扩展、格式→清理。不可修的（标记给用户）：需要真实实验数据、需要领域专家判断、需要用户决定、需求自相矛盾。

**4. 占位数据** — 审查通过后或第 2 轮起只有不可修问题：生成占位表格/图表，标注 `> **⚠️ 占位数据，需要替换为真实结果**`，使用合理的数值范围，标注真实实验会产生什么。

**5. 决定下一步** — 交付改进后的章节并列出证据或作者决策缺口。可以建议再做一轮，但不要自动开始。

### 可选摘要

```
轮次：N。章节：[列表]。已处理问题：[数量]。
剩余：[不可修] 第 X 节 — 描述。
下一步：[用户应该做什么]。
```

## 中文写作规范

写作时遵守 `plain-chinese-writing` 技能中的相关规范。以下是要点提醒。

### 标点规则
- 破折号完全禁用
- 冒号限制使用（列举三项以上、引出定义时可用）
- 括号限制使用（缩写定义、编号可保留）
- 黑体限制使用（关键词定义、列表标签可保留）

### 语言风格
- 平直、准确、简洁、自然
- 多用动词，少抽象名词
- 语义完整，长短相间
- 注意中文特征：主题突出、意合衔接、主动优先
- 不为了规避标点规则而机械拆句
- 不把所有段落写成相同节奏
- 直接写具体困难、原因和后果，少用模板化转折
- "的"字不要连用三个以上

### 翻译腔和 AI 味
- 不堆叠名词，不滥用"进行""实现""基于"
- 不用英文语序，不用过多框式介词
- 不写空泛开头、泛化评价、套话结尾
- 不机械使用"不在于……而在于……"转折模板

## 规则

1. 只在第 6 级标题写内容。主题≤50字符。正文长度服从语义完整和章节需要，通常不超过 800 字。不要修改结构标题。
2. 有引用就引用，没有就标注 `[需要引用]`。
3. 不捏造数据。占位数据标注 `> **⚠️ 占位数据，需要替换为真实结果**`。
4. 每个问题引用原文和位置。一个问题一条。
5. 标记缺失前先搜索确认确实缺失。

## 范围建议

- 默认只做一轮。
- 用户明确要求继续迭代时再做下一轮。
- 每轮最多处理五个章节，便于核对。
- 不可修问题写入局限说明，不触发自动重试，也不阻止其他范围内的工作。
