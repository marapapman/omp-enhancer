# 中文去 AI 痕迹工具

扫描中文文档中的 AI 写作痕迹并建议自然替换。逐项处理，每项用户确认后应用。最后给出修改前后对比。

核心原则：宁缺毋滥。如果文本已经自然、严谨且没有明显 AI 特征，保留原文，不要为了修改而修改。

防副作用原则：去 AI 味必须证据驱动。只有出现空泛开头、套话结尾、泛化评价、翻译腔、机械转折、机械列举或过度结构提示时才修改。不得把所有句子改短，不得把所有段落改成同一种节奏。破折号完全禁用。冒号、括号、黑体限制使用但不等于机械拆句。处理限定、转折和解释信息时，应优先保留语义完整，再调整到句首、句尾或并入主句。只有句子确实过长或层级混乱时，才拆成独立句子。

写作规范详见 `/skill:plain-chinese-writing`，本技能在此基础上专注于 AI 痕迹扫描与替换。

## 触发场景

用户要求"去 AI 味""去翻译腔""让文字更自然""改得更像人写的""润色中文"等。也适用于审查输出时发现文档过于模板化。

## 扫描模式

对每个模式，给出原文、问题诊断和建议替换。

### 1. 词汇规范化（意图驱动）

凡是无实质信息量的情感渲染性表达，或试图通过华丽辞藻掩盖逻辑空洞的词汇，均应替换为具体、客观的学术描述。

| 原文 | 问题 | 替换为 |
|------|------|--------|
| "毋庸置疑" | 情感渲染 | 删除或改为具体陈述 |
| "不可磨灭的贡献" | 过度拔高 | 具体说明贡献是什么 |
| "范式转移" | 花哨 | "转变"或具体说明变化 |
| "颠覆性" | 夸大 | "新的"或具体说明不同之处 |
| "令人惊叹" | 情感渲染 | "显著"或给出具体数据 |
| "为了解决这一痛点" | 互联网黑话 | "针对上述问题" |
| "展现了令人惊叹的能力" | 夸大 | "表现出显著的性能提升" |
| "深刻" | 空泛 | 具体说明深刻在哪 |
| "切中要害" | 花哨 | "准确" |
| "本质"（非哲学语境） | 空泛 | "核心"或具体说明 |

保持核心专业术语的准确性，绝对不要为了"去 AI 味"而随意替换领域内的专有名词。

### 2. 句式与结构自然化（去翻译腔与机械感）

| 模式 | 检查什么 | 修改前 → 修改后 |
|------|----------|-----------------|
| **英式长定语** | "一个…的…的…"连续修饰结构 | "一个面向大规模数据的分布式处理系统" → "一个分布式处理系统，专门面向大规模数据" |
| **被动语态过多** | 中文学术写作少用"被"字句 | "…被用来优化…" → "采用…优化…" |
| **机械列举** | "首先…其次…最后…"或"1. 2. 3." | 融合成逻辑连贯的段落。但若列举在当前语境下逻辑更清晰，如陈述算法核心步骤，可酌情保留 |
| **机械转折** | "因此，问题不在于……而在于……"，"核心困难不在于……而在于……" | 只有需要纠正误解时才保留对照句。多数情况下直接写具体困难、原因和后果 |
| **过度使用"进行"** | "对……进行分析" | "对数据进行了清洗" → "清洗了数据" |
| **过度使用"实现"** | "实现了性能提升" | "提高了性能" |
| **过度使用"基于"** | "基于上述分析" | "根据上述分析" |
| **名词堆叠** | 连续多个名词的偏正结构 | "数据驱动的基于深度学习的智能分析系统" → "用深度学习分析数据的系统" |
| **抽象主语过多** | 主语是抽象名词短语 | "实验结果的提升表明了模型设计的有效性" → "实验结果表明模型设计有效" |
| **框式介词过多** | "在……方面""对于……来说""关于……的问题" | 能省则省，直接说 |
| **"的"字连用** | 连续三个以上的"的" | 重组句式 |

### 3. 空泛开头

| 原文 | 替换为 |
|------|--------|
| "近年来，随着人工智能技术的快速发展……" | 直接说问题或研究对象 |
| "在当今数字化时代……" | 直接说背景中最具体的一点 |
| "随着社会的不断进步……" | 删除，直接进入内容 |
| "在复杂多变的现实环境中……" | 直接说具体环境 |
| "随着……的不断发展……" | 直接说事实 |

### 4. 过度结构提示

| 原文 | 替换为 |
|------|--------|
| "本文将从以下几个方面展开。" | 删除，直接进入内容 |
| "接下来，我们将深入探讨。" | 删除 |
| "综上所述，" | 删除或改为一句总结 |
| "总而言之，" | 删除或改为一句总结 |
| "本文旨在深入探讨……" | "本文研究……"或"本文提出……" |

### 5. 泛化评价

| 原文 | 替换为 |
|------|--------|
| "该方法具有重要意义。" | 具体说明意义是什么 |
| "发挥着重要作用。" | 具体说明起什么作用 |
| "提供了有益参考。" | 具体说明参考什么 |
| "具有广阔前景。" | 说明可用于什么地方 |
| "展现出巨大潜力。" | 给出具体数据或结果 |

### 6. 套话结尾

| 原文 | 替换为 |
|------|--------|
| "未来仍需进一步探索。" | 写具体的后续工作 |
| "相信该方法将在更多领域发挥价值。" | 删除或写具体的应用方向 |

### 7. 格式问题

| 模式 | 检查 | 处理 |
|------|------|------|
| **破折号** | 完全禁用—— | 保留原意后改写为普通句式。必要时拆句 |
| **冒号滥用** | 非列举、非定义场景下的冒号 | 改为自然承接句。列举三项以上或引出定义时可保留 |
| **括号滥用** | 括号内塞了长句或完整解释 | 把补充信息并入主句、放到句首或句尾。缩写定义和必要编号可保留 |
| **黑体滥用** | 用黑体代替表达强度 | 删除黑体标记。关键词首次定义或列表标签可保留 |
| **插入语** | 句中频繁插入解释 | 保留限定信息，调整到句首、句尾或并入主句。必要时拆句 |
| **Markdown 标记** | Word 粘贴场景中的 `**``*``#` | 删除所有 Markdown 标记 |

### 8. 对称段落

每段开头一样、长度一样、结构一样时，才判断为对称段落。处理时可以合并短段落、改写相同开头、变化句长，但不要为了变化而破坏原有论证顺序。

## 处理流程

1. **扫描。** 通读全文。识别上述模式的每一个实例。收集为发现列表，附原文引用。

2. **展示发现。** 对每个发现，展示：
   - 模式名称
   - 原文（附上下文，需要时前后各约 2 行）
   - 问题诊断（具体说明为什么这是 AI 痕迹）
   - 建议替换
   - 问："是否替换？（y/n）"

3. **应用。** 用户确认后，应用替换。

4. **输出对比。** 所有发现处理完后，展示：
   - **修改前** — 完整原文
   - **修改后** — 完整修改后文本
   - **变更统计** — 按模式类型统计替换数量

## 修改阈值（关键）

- 如果文本已经自然、严谨且没有明显 AI 特征，保留原文
- 对于高质量的输入，给出明确肯定："原文表达严谨自然，无明显 AI 痕迹，建议保留。"
- 不要为了换词而换词
- 不要把自然的中长句全部改成短句
- 不要机械使用"因此……不在于……而在于……"这类转折模板
- 不要删除原文中的限定、转折、因果和承接信息

## 输出格式

```
--- 发现 1 / N ---
模式：[模式名称]
原文：[带上下文的原文引用]
问题：[具体说明为什么这是 AI 痕迹]
建议：[替换文本]
是否替换？（y/n）
```

所有发现处理完后：

```
--- 修改前 ---
[完整原文]

--- 修改后 ---
[完整修改后文本]

--- 变更统计 ---
- 翻译腔：3 处替换
- AI 味：2 处替换
- 套话：1 处替换
...
```
