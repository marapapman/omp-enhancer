# 中文论文润色技能

对中文论文段落进行专业审视与润色。核心原则：尊重原著，克制修改。

## 核心原则

**修正阈值是生死线。**

- 必须修改：仅在检测到口语化表达、语法错误、逻辑断层或严重翻译腔时才修正
- 禁止修改：如果原文逻辑通顺、用词准确，严禁为了追求形式变化而强行替换同义词或重组句式
- 保持作者原有的行文风格是第一优先级
- 不为了规避标点规则而机械拆句。处理限定、转折和解释信息时，优先保留语义完整，再调整位置或句式
- 避免机械转折。不要反复使用"因此……不在于……而在于……"这类模板，除非原文确实需要纠正误解

写作规范详见 `/skill:plain-chinese-writing`，本技能在此基础上专注于润色修正。

## 适用场景

- 用户要求"润色这段中文""修改一下措辞""优化表达"等
- 中文论文各章节的终稿打磨
- 用户提供了中文草稿需要整理为正式学术文本

## 工作流程

### 第一步：读取原文

用 `read` 读取待润色的文档或段落。理解完整的上下文和作者的论证意图。

### 第二步：逐段审查

对每一段落，按以下优先级检查：

**必须修改的（优先级高）：**
1. 口语化表达（"我们觉得"→"实验结果表明"，"效果变好了"→"性能显著提升"）
2. 语法错误
3. 逻辑断层或论证跳跃
4. 严重翻译腔（英式长定语、名词堆叠、过度被动句、框式介词过多）
5. AI 味明显的表达（空泛开头、泛化评价、套话结尾、机械转折）
6. 破折号（完全禁用，必须改写）

**不修改的（即使可以改得更好）：**
- 原文逻辑通顺、用词准确的句子
- 作者特有的行文风格（只要不影响理解）
- 可以换一个同义词但原文也不错的选词
- 合理使用的冒号、括号、黑体

### 第三步：语体规范

检查以下语体问题：
- 是否存在陈旧公文腔（无故用"拟"代"旨在"，用"系"代"是"）→ 保持现代学术风
- 连接词是否机械堆砌 → 优先依赖语序自然衔接
- 段落是否有清楚主线 → 保留必要的限定、证据、转折和小结，不要求每段只能有一个信息点
- 中英文混排空格是否规范（中英文之间加空格）

### 第四步：输出

对每个段落输出两部分：

```
Part 1 [润色文本]
[如果进行了润色：输出润色后的文本]
[如果原文无需修改：直接原样输出原文]

Part 2 [审查意见]
[如果进行了润色：简要说明修改点（例如：修复了指代不明，去除了口语表达）]
[如果原文无需修改：直接给出肯定评价（例如："原文逻辑清晰，表达规范，符合出版要求，未做修改。"）]
```

### 第五步：应用

用户确认润色结果后，用 `edit` 将修改写入文档。

## 格式要求

- 输出纯文本，不使用 Markdown 加粗、斜体（除非原文有合理用途）
- 使用中文全角标点符号
- 数学符号或英文术语周围保留合理空格
- 保留关键技术名词，不强行翻译英文术语

## 反面模式

- 不要为了刷存在感而修改原本通顺的句子
- 不要把"可改可不改"的风格问题当必须修改
- 不要把"换个词听起来更高级"当作修改理由
- 不要统一化所有段落风格，保留作者的语言个性
- 不要把自然的中长句全部改成短句
- 不要把每个关键判断都改成"不是……而是……"或"不在于……而在于……"
- 不要把合理使用的冒号、括号、黑体当成问题

## 自检清单

输出前逐项确认：
1. 我修改的部分是否都是必要的，存在明显问题的？
2. 如果我没改动，是否完整输出了原文并给予了肯定？
3. 我是否为了追求形式变化而改了本不该改的句子？
4. 破折号是否全部去除？
5. 冒号、括号、黑体的修改是否确实必要？
