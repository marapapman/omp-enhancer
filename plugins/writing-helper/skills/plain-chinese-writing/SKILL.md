# 通用中文平直写作 Skill

## 0. 适用范围

本技能不是可选的润色工具，而是生成中文文字时必须遵守的写作规范。

任何时候，只要输出中文，就要按本技能的要求写。

这包括以下所有场景。

- 回答用户问题
- 生成文档
- 修改文字
- 写邮件
- 写报告
- 写论文
- 写技术说明
- 写项目文档
- 写申请材料
- 写公众号文章
- 写代码注释
- 写 README
- 任何需要输出中文的场景

收到用户指令时先问自己：我要说的内容用这个 Skill 该怎么写。然后直接按本 Skill 的要求写出来。

## 1. 使用场景

本技能在以下情况下自动生效，不需要用户额外提醒。

- 生成新的中文文字
- 修改已有的中文文字
- 回复用户的中文问题
- 输出任何形式的中文内容

用户也可以主动要求以下操作，此时也要遵守本技能。

- 去掉翻译腔
- 去掉 AI 味
- 改得更自然
- 改得更平实
- 改得更直白
- 规范中文表达
- 避免套话和花哨语言
- 润色中文文字

本技能适用于以下文本类型。

- 学术论文
- 学位论文
- 研究报告
- 技术文档
- 项目说明
- 邮件

- 公众号草稿
- 新闻稿
- 内部沟通材料
- 日常说明文字
- 对话回复
- 代码注释

## 2. 核心目标

不管是写新文字还是修改旧文字，输出都应满足以下要求。

1. 没有翻译腔。
2. 没有 AI 味。
3. 不堆概念。
4. 不故作高级。
5. 不使用花哨表达。
6. 不使用不必要的成语。
7. 不使用明显模板化结构。
8. 语义完整，句子长短相间。
9. 逻辑清楚。
10. 读起来像中文母语者自然写出的文字。
11. 平直、清楚、自然。

### 2.1 防止机械化改写

硬性限制规则不等于短句化，也不等于统一段落节奏。

修改文字时，先保留原文的主线、限定、转折、承接和作者语气。只有在句子确实过长、层级混乱、存在翻译腔或影响理解时，才拆句或重组。

不要为了规避规则而机械拆句。不要把自然的限定信息删掉。不要把每段都改成相同的短句结构。

避免机械转折。不要反复使用"因此，问题不在于……而在于……""核心困难不在于……而在于……""不是……而是……"这类模板。只有文本确实需要排除一种误解并强调另一点时，才使用对照句。多数情况下应直接说具体困难、原因和后果。

去 AI 味必须有明确证据。只有出现空泛开头、套话结尾、泛化评价、翻译腔、机械转折、机械列举或过度结构提示时才修改。原文自然、准确、连贯时必须保留。

### 2.2 保留语义锚点

修改已有文本前，记录频率和强度限定、模态、范围、否定、比较和因果方向、数字和单位、引用和标识符，以及 LaTeX 数学、交叉引用、命令和结构。这些都是语义锚点，不是可以为了简洁或去 AI 味而删除的填充词。除非用户或证据明确要求，否则保持不变。修改后只比对一次。任务只读时，在最终响应中返回修改建议，不创建工作流文件。

## 3. 标点和格式规则

### 3.1 破折号——完全禁用

任何时候都不要使用破折号（——）。

错误示例

```text
这项工作有一个明显问题——数据来源不稳定。
```

修改为

```text
这项工作的一个明显问题是数据来源不稳定。
```

或

```text
这项工作存在一个明显问题。数据来源不稳定。
```

### 3.2 冒号——限制使用

冒号在中文中有合理用途，但不要滥用。以下情况可以使用冒号。

- 列举前三项以上内容时。"原因有三个：数据少、噪声大、标注不一致。"
- 引出定义或公式时。
- 技术文档中的键值对或参数说明。

以下情况不要使用冒号，改用自然句式。

- 单纯引出一句话。"原因是：数据不够。" → "原因是数据不够。"
- 在段落中间机械地用冒号分层次。
- "步骤如下：……" → 直接写步骤。

### 3.3 括号——限制使用

括号在以下情况可以保留。

- 学术论文中首次出现英文缩写时。"大语言模型（LLM）在文本生成任务上表现突出。"
- 必要的数学符号、公式编号、法规编号。
- 简短的补充限定，不影响主句节奏。

以下情况不要使用括号。

- 括号里塞了长句或完整解释。把内容移出来，单独成句或并入主句。
- 连续多个括号打断阅读节奏。
- 用括号替代正常的句子结构。

### 3.4 插入语——限制使用

不要频繁在句中插入解释。处理插入信息时，先保留限定、转折和承接，再调整位置。可以放到句首、句尾，或并入主句。只有在句子确实过长或层级混乱时，才拆成独立句子。

错误示例

```text
这个方法在训练数据不足的情况下，尤其是在低资源场景中，仍能保持稳定效果。
```

修改为

```text
这个方法在训练数据不足时仍能保持稳定效果。在低资源场景中也是如此。
```

### 3.5 黑体强调——限制使用

Markdown 黑体不是不能用，但不要滥用。

可以使用黑体的场景。

- 文档中的关键词或术语首次定义。
- 列表中的标签词。
- 必要的视觉区分。

不要使用黑体的场景。

- 代替正常的表达强度。"这一步 **非常重要**。" → "这一步很重要。"
- 频繁加粗导致页面花哨。
- 用黑体替代清晰的逻辑表达。

## 4. 中文语言特征——正面指导

### 4.1 主题突出

中文是主题突出语言，不是主语突出。很多中文句子以话题开头，后面跟评论或说明。这是自然的中文表达，不需要硬套英文的主谓宾结构。

自然写法

```text
这个问题，目前没有好的解决办法。
性能方面，新方法有明显提升。
已有的方法，大多只关注准确率。
```

不要写成

```text
关于这个问题，目前不存在有效的解决办法。
在性能方面，新方法实现了显著的提升。
对于已有的方法而言，它们大多只关注准确率。
```

后一种写法多了"关于""在……方面""对于……而言"这类框式介词，是典型的英文影响。

### 4.2 流水句和意合

中文句子之间不总是需要显式连词。靠语序和语义衔接，一句接一句往下说，是自然的中文节奏。这种写法叫"意合"。

自然写法

```text
实验在三个数据集上进行，每个数据集划分训练集和测试集，比例 8:2。模型训练 50 个 epoch，学习率初始值 0.001，每 10 个 epoch 减半。
```

不要硬塞连词

```text
实验在三个数据集上进行，并且每个数据集被划分为训练集和测试集，其比例为 8:2。此外，模型训练 50 个 epoch，同时学习率初始值设为 0.001，而且每 10 个 epoch 减半。
```

### 4.3 中文衔接词

中文有自己的衔接手段，不要依赖英文连词的直译。

| 英文连词 | 不要写成 | 可以用 |
|----------|----------|--------|
| however | 然而、不过、但是 | 但、不过、却 |
| therefore | 因此、所以 | 所以、于是、就 |
| in addition | 此外、除此之外 | 还有、同时、也 |
| moreover | 而且、此外 | 又、还 |
| in other words | 换言之 | 也就是说、即 |
| specifically | 具体来说 | 具体地、确切地说 |
| that is to say | 也就是说 | 就是 |

要点：能用单字连词就不用双字连词，能用短词就不用长词。

### 4.4 的/地/得

- **的** 连接定语和名词。"这是好的方法。"
- **地** 连接状语和动词。"逐步地提高准确率。"
- **得** 连接动词和补语。"效果提升得很明显。"

注意避免"的"字连用过多。

```text
这是一个基于深度学习的用于文本分类的模型。
```

改为

```text
这个模型用深度学习做文本分类。
```

### 4.5 主动优先

中文天然偏向主动句。能用主动句就不要用被动句。

```text
该方法被用于图像分类。  →  该方法用于图像分类。
数据被分为训练集和测试集。  →  数据分为训练集和测试集。
```

但不要为了主动化而硬改。有些句子用"被"更清楚，可以保留。

```text
这个问题被多次讨论过。  ←  可以保留
```

## 5. 中英文混排规范

### 5.1 术语处理

- 中文已有通用译名的，用中文。"梯度下降""反向传播""注意力机制"。
- 中文译名不统一或生硬的，直接用英文。transformer、fine-tuning、token、embedding。
- 英文术语首次出现时，可以在括号里标注英文原名，帮助读者对应。
- 同一篇文档中对同一个术语保持一致。不要一段写"预训练"另一段写"pre-training"。

### 5.2 空格规范

- 中文和英文之间加一个空格。"使用 BERT 模型"而不是"使用BERT模型"。
- 中文和数字之间加一个空格。"准确率提高了 15%"而不是"准确率提高了15%"。
- 英文和中文标点之间不加空格。
- 全角标点和任何字符之间都不加空格。

### 5.3 引用英文文献时的表述

不要把英文论文的论证结构原样翻译过来。英文论文喜欢"先行词 + 定语从句 + 总结"的三层结构，中文不必照搬。

英文写法的直译

```text
Smith 等人（2023）提出了一种方法，该方法通过引入注意力机制来解决长序列建模的问题，从而在多个基准测试上取得了显著的性能提升。
```

改为自然的中文

```text
Smith 等人（2023）在模型中引入注意力机制来处理长序列。他们在多个基准测试上取得了明显提升。
```

## 6. 翻译腔检查清单

发现以下表达时，应改成自然中文。

### 6.1 名词堆叠

```text
该模型实现了对用户行为模式的有效识别。
```

改为

```text
该模型能识别用户行为模式。
```

### 6.2 过度使用"进行"

```text
我们对数据进行了清洗。
```

改为

```text
我们清洗了数据。
```

### 6.3 过度使用"实现"

```text
该方法实现了性能提升。
```

改为

```text
该方法提高了性能。
```

### 6.4 过度使用"基于"

```text
基于上述分析，本文提出一种方法。
```

改为

```text
根据上述分析，本文提出一种方法。
```

或

```text
上述分析说明需要一种新的方法。本文提出一种方法。
```

### 6.5 英文语序

```text
这是一个被广泛使用的方法。
```

改为

```text
这个方法应用较广。
```

### 6.6 抽象主语过多

```text
实验结果的提升表明了模型设计的有效性。
```

改为

```text
实验结果表明模型设计有效。
```

### 6.7 被动句过多

```text
该问题被本文进一步研究。
```

改为

```text
本文进一步研究这个问题。
```

### 6.8 框式介词过多

"在……中""在……方面""对于……来说""关于……的问题"这类框式介词是英文 of/in/for/with 的直译痕迹。能省则省。

```text
在模型训练的过程中，在特征提取方面，存在着不足。
```

改为

```text
模型训练时的特征提取有不足。
```

### 6.9 "的"字连用

连续三个以上的"的"需要重组。

```text
这是一个基于深度学习的面向文本分类的自注意力模型。
```

改为

```text
这个自注意力模型用深度学习做文本分类。
```

## 7. AI 味检查清单

发现以下表达时，应删除或改写。

### 7.1 空泛开头

避免使用。

```text
近年来，随着人工智能技术的快速发展……
在当今数字化时代……
随着社会的不断进步……
在复杂多变的现实环境中……
```

改法。

- 直接说问题。
- 直接说研究对象。
- 直接说背景中最具体的一点。

示例。

```text
大语言模型已经被用于文本生成、问答和代码生成。
```

### 7.2 过度提示结构

避免使用。

```text
本文将从以下几个方面展开。
接下来，我们将深入探讨。
综上所述。
总而言之。
```

改法。

```text
本文讨论三个问题。
```

或直接进入内容。

### 7.3 泛化评价

避免使用。

```text
具有重要意义。
发挥着重要作用。
提供了有益参考。
具有广阔前景。
展现出巨大潜力。
```

改法。

写清楚意义、作用、参考和潜力具体是什么。

错误示例

```text
该方法具有重要意义。
```

修改为

```text
该方法减少了人工标注时间。
```

### 7.4 套话结尾

避免使用。

```text
未来仍需进一步探索。
相信该方法将在更多领域发挥价值。
```

改为具体表述。

```text
后续工作可以在更大的数据集上测试该方法。
```

### 7.5 机械转折

避免把论证写成固定对照句。

```text
因此，这一场景的核心困难不在于覆盖率是否具有指导价值，而在于能否以低开销、稳定的方式获得覆盖信息。
问题不在于是否需要数据，而在于如何获得数据。
关键不在于模型是否足够大，而在于数据是否足够干净。
```

这些句式不是绝对不能用，但不能作为默认转折模板。只有需要纠正读者误解时，才使用"不在于……而在于……"这类句式。

改法。

- 直接说困难是什么。
- 直接说前一个条件已经成立。
- 用因果和承接代替模板化对照。
- 保留必要转折，但不要把转折写成口号。

示例。

```text
覆盖率可以指导测试，但获取覆盖信息的成本较高，结果也容易受运行环境影响。所以，这一场景更需要低开销、稳定的覆盖采集方法。
```

## 8. 用词规范

### 8.1 绝对禁用词

这些词在任何中文写作中都不使用。

```text
赋能
抓手
闭环
沉淀
落地
生态
矩阵
路径（非字面义）
维度（非数学义）
场景化
体系化
全面提升
持续推进
深度融合
显著增强
充分发挥
有机结合
```

### 8.2 需要检查的词

这些词可以使用，但每次使用都要确认是否具体、必要。

```text
重要意义
关键作用
积极推动
有效促进
不断优化
持续深化
深入探讨
全面分析
多维度
复杂性
重要性
创新性
鲁棒性
可解释性
显著性
前沿
范式
图景
蓝图
脉络
生态系统
```

如果可以用更具体的说法替代，优先替代。

### 8.3 学科术语不限制

以下词汇是学科标准术语，不需要替换。

```text
显著性检验
鲁棒性分析
范式转换
统计分析
梯度下降
注意力机制
```

只有在这些词被用于非专业语境的泛泛之谈时，才需要检查。

### 8.4 成语和花哨表达

尽量不用成语和修辞性词语。

示例。

```text

至关重要
日益凸显
方兴未艾
层出不穷
如火如荼
蓬勃发展
深远影响
保驾护航
添砖加瓦
```

改为普通表达。

```text
重要
越来越明显
正在发展
很多
影响较大
提供支持
```

## 9. 写作步骤

### 9.1 写新文字时

每次需要输出中文时，直接按以下步骤写出。

第一步，确定要说什么。把核心信息记下来。

第二步，用自然的话把信息写出来。每句话只承担清楚的表达任务，但不要机械拆句。注意中文的特征：主题突出、意合衔接、主动优先。

第三步，检查标点和格式。

- 去掉破折号。
- 冒号只在列举三项以上或引出定义时使用。
- 括号只用于缩写定义和必要的补充。
- 插入语不要频繁使用。
- 黑体只用于必要的关键词标注。
- 换成不用成语的说法。
- 只拆确实过长或层级混乱的句子。
- 抽象词换成具体词。
- 名词结构换动词结构。
- 检查"的"字是否连用过多。

第四步，读一遍，确认通畅。

### 9.2 修改已有文字时

第一步，判断文本用途。

- 学术论文
- 报告
- 邮件
- 项目说明
- 公众号文章
- 申请材料
- 日常文字

不同文本保留不同正式程度。

第二步，找出主要问题。

- 翻译腔
- AI 味
- 套话
- 花哨表达
- 成语堆叠
- 名词堆叠
- 过长或层级混乱的句子
- 破折号
- 冒号、括号、黑体的滥用
- "的"字连用

第三步，重写。

- 不用破折号。
- 冒号、括号、黑体限制使用。
- 尽量不用成语。
- 只拆确实过长或层级混乱的句子。
- 抽象表达改具体。
- 名词结构改动词结构。
- 删除不提供信息的句子。
- 注意主题突出和意合衔接。

第四步，自检。

- 是否还有破折号。
- 冒号、括号、黑体是否必要。
- 是否还有明显套话。
- 是否还有翻译腔。
- 是否还有 AI 味。
- 是否存在影响理解的长句。
- 是否出现机械短句化。
- 是否出现机械转折。
- "的"字是否连用过多。

## 11. 特殊情况

### 11.1 学术论文

保留必要的专业准确性。不要为了口语化牺牲概念。

可以保留。

- 必要术语
- 必要引用格式
- 必要公式编号
- 必要英文缩写（用括号标注）
- 必要的冒号（列举、定义）

但要避免。

- 过度宏大叙述
- 空泛意义表述
- 过度结构化提示
- 过度使用"本文旨在""本文聚焦于""本文从多个维度"

### 11.2 技术文档

优先清楚，不追求文采。

示例。

```text
运行脚本前，先确认配置文件已经生成。
```

不要写成。

```text
在运行脚本之前，需要对配置文件的生成情况进行确认。
```

### 11.3 邮件和沟通文字

语气自然、礼貌、直接。

示例。

```text
我明天下午前把修改版发给你。如果你希望我先改摘要，也可以告诉我。
```

不要写成。

```text
我将于明日下午前完成相关修改工作，并同步给您。如您有优先处理事项，也欢迎进一步指示。
```

### 11.4 宣传稿和公众号文章

可以稍微有节奏，但仍应避免空话和夸张。

不要使用。

```text
重磅来袭
精彩纷呈
干货满满
不容错过
```

改为具体信息。

```text
活动将在周五晚上举行，内容包括报告和问答。
```

## 12. 质量标准

一段合格的修改后文本应满足。

1. 普通中文读者能顺畅读完。
2. 没有明显机器生成痕迹。
3. 没有英文直译语序。
4. 没有空泛套话。
5. 没有不必要的修饰词。
6. 没有破折号。
7. 冒号、括号、黑体使用克制且必要。
8. 句子长度适中，长短相间。
9. 信息比原文更清楚。
10. 语气不夸张。
11. 不牺牲原意。

## 14. 最终检查清单

输出前逐项检查。

- 文本中没有破折号（——）。
- 冒号的使用是必要的（列举、定义）。
- 括号的使用是必要的（缩写、编号）。
- 黑体的使用是克制的。
- 文本中没有明显成语。
- 文本中没有"值得注意的是""综上所述""本文旨在深入探讨"等套话。
- 文本中没有机械使用"因此……不在于……而在于……"这类转折模板。
- 文本中没有不必要的"进行""实现""基于"。
- 文本中没有过多的框式介词"在……方面""对于……来说"。
- "的"字没有连用三个以上。
- 过长或层级混乱的句子已经整理，语义完整。
- 原意没有丢失。

## 15. 最重要的规则

以下是本 Skill 最重要的规则。每次写中文前先确认这几条。

1. 不要用破折号。
2. 冒号、括号、黑体限制使用，不滥用。
3. 不要用成语和花哨语言。
4. 不要用插入语打断句子节奏。
5. 不要用翻译腔句式。
6. 不要用"值得注意的是""综上所述""本文旨在深入探讨"等套话。
7. 不要机械使用"因此……不在于……而在于……"这类转折模板。
8. 注意中文特征：主题突出、意合衔接、主动优先。
9. 语义完整，句子长短相间。
10. 表达直接一些，但不要机械短句化。

这些规则在写新文字和修改旧文字时都要遵守。
