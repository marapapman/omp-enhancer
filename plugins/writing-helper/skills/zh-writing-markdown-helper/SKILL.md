# 中文逐段写作技能

逐段写中文 markdown 文档。写一段，等用户确认，再写下一段。为结构化中文论文设计，配合中文平直写作规范。

## 适用场景

- 用户要求逐段起草结构化中文文档
- 用户要求一段一段写，每段确认
- 写论文、报告、技术说明等中文文档

## 文档结构

- `##` 到 `#####`：章节标题，不要修改
- `######`：段落节点。标题=主题句（≤50字符），正文长度服从语义完整和章节需要
- HTML 注释 `<!-- description: ... -->`：描述每节应覆盖的内容

## 输入文件

| 文件 | 用途 |
|------|------|
| paper.md | 主文档，扫描空章节，读取上下文 |
| storyline.md | 研究叙事、核心想法、方法 |
| .pi/research/literature.md | 文献摘要 |
| .pi/research/papers/ | 下载的论文，只读当前段落相关的 |

## 逐段写作流程

### 第一步：生成整体大纲

1. 读取 storyline.md 和已有 paper.md 的结构（第 2-5 级标题及其 description 元数据）。
2. 为每个空的第 5 级章节提出主题句（每个≤50字符），列在父标题下面。
3. 把大纲展示给用户。问："这是段落大纲，是否接受？需要修改还是重新生成？"
4. 等用户回复，不要自作主张。

### 第二步：定位并提议段落

1. 从上到下读 paper.md。找到第一个没有第 6 级子节点的第 5 级章节。
2. 报告章节名称和 description 元数据。问："下一个小节是[标题]。是否在这里写一段？"
3. 等用户确认。

### 第三步：收集上下文并起草

读取这些文件获取上下文：
- storyline.md
- .pi/research/literature.md
- 相关的 .pi/research/papers/ 文件
- 当前 paper.md（避免重复）

然后写一段（一个 `######` 节点）：
- 标题：大纲中批准的主题句（≤50字符）
- 正文：支撑文字，长度服从语义完整和章节需要，需要时加引用
- 不要修改第 2-5 级标题
- 用平直、准确的中文。简洁。不凑字数。不要机械拆句。

### 第四步：用户审查

问："接受、修改（给反馈）还是重写？"

等用户回复。

### 第五步：应用并循环

- 接受：用 edit 把新的 `######` 节点插入目标章节下面。回到第二步。
- 修改：按用户反馈修改，重新展示（回到第四步）。
- 重写：丢弃重写（回到第三步）。

没有用户明确确认，不要自动进入下一段。

## 中文写作规范

写作时遵守 `/skill:plain-chinese-writing` 的完整规范。以下是要点提醒。

### 标点规则
- 破折号完全禁用
- 冒号限制使用（列举三项以上、引出定义时可用）
- 括号限制使用（缩写定义、编号可保留）
- 黑体限制使用（关键词定义、列表标签可保留）

### 语言风格
- 平直、准确、简洁
- 多用动词，少用抽象名词
- 语义完整，长短相间
- 注意中文特征：主题突出、意合衔接、主动优先
- 不为了规避标点规则而机械拆句
- 保留必要的限定、转折和承接信息
- 不机械使用"不在于……而在于……"转折模板
- 直接写具体困难、原因和后果，少用模板化转折
- "的"字不要连用三个以上

### 翻译腔和 AI 味
- 不堆叠名词，不滥用"进行""实现""基于"
- 不用英文语序，不用过多框式介词
- 不写空泛开头、泛化评价、套话结尾

## 反面模式

- 不要一次写多段
- 不要跳过大纲生成
- 不要在用户确认前插入内容
- 不要修改章节标题
- 不要把所有段落写成相同的短句结构
- 不要把每个关键判断都写成"不是……而是……"或"不在于……而在于……"
