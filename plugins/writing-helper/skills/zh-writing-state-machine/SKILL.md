# 中文严格写作状态机

中文严格写作采用读取来源、拟稿、自检和交接的证据矩阵。只有用户明确要求段落上下文隔离时，才在每段后重置。每条引用、每个数字和事实论点都应有可观察来源。章节主线、术语定义和用户已批准的写作风格可以继承，避免段落之间割裂。

## 工作流边界

只在 Main 选择工作流 `writing.zh`、加载其精确 workflow reference 和本 Skill，再把任务派给 `zh-writer` 子 Agent 后使用本 Skill。这是受派 writer 子 Agent 的有界局部方法，不选择或调度 Agent。不要递归 fork、spawn 或 delegate。Main 保留父级 TODO、集成、最终验证和面向用户的交付权。

证据矩阵只是 writer 局部自检，不能满足或替代独立 `zh-checker` delivery。只做一次有界局部处理；不启动自动修复循环，也不创建完成门禁。把段落、矩阵、来源和未解决缺口返回给 Main。

writer 子 Agent 始终只交付建议稿。返回完整建议文本；有界补丁更清楚时，使用
SEARCH/REPLACE 块或 unified diff。Main 保留权限决策和实际文件修改。不要创建或
持久化目标文档、研究产物或 review log。

## 适用场景

- 写实验结果、数据章节、或任何含引用和数值论点的内容
- 用户明确要求"严格模式"或"无幻觉"写作
- 多段落文档中段落间不允许上下文漂移

## 核心流程：每段一个独立循环

用户明确要求严格段落隔离时，每段独立处理。其他情况下，把本流程作为质量建议，不强制每段开启新会话。

### 1. 读取 — 只读当前需要的

只读当前这一段需要的源数据。事实证据只依赖当前段落的来源。可以读：
- 目标文档（扫描章节结构、术语定义和相邻段落的衔接）
- 当前段落的源文档、数据表或引用文献
- 章节的元数据/描述（如果有）

只有用户或 host 授权网络访问且存在实时网络能力时，才使用网络。本地证据不足而任一条件不满足时，不要联网，把来源缺口返回给 Main。

### 2. 写作 — 恰好一段

输出一段，结构如下：

```markdown
###### 标题 — 主题句（≤50字符）

正文长度服从语义完整和证据表达，含正确的 `[@引用]` 标记。
```

约束：
- 标题 ≤50 字符
- 正文语义完整，不为了变短而拆散论证
- 引用必须精确，从来源复制，不要改写引用标签
- 来源元数据和建议的 review-log 条目随结果交付。不要在文档中用 HTML 注释

### 3. 自检 — 建议验证矩阵

| 检查项 | 通过/失败 |
|--------|-----------|
| 标题 ≤50 字符 | — |
| 正文语义完整，没有机械短句化 | — |
| 正文中每个 `[@...]` 引用在来源中存在 | — |
| 交付结果或已授权的 review log 中有来源真实路径 | — |
| 没有超出源数据支持的论点 | — |
| 没有翻译腔表达 | — |
| 没有 AI 味表达 | — |
| 没有破折号 | — |
| 冒号、括号、黑体使用合理 | — |

在这一轮受派 writer 任务内，修复能从现有证据解决的检查项。无法解决的项目写入局限说明，并只继续任务指定的范围。

### 4. 交接 — 遵守操作授权

把完整段落建议和有界插入补丁返回 Main。是否在获授权的集成阶段持久化，由 Main 决定。

### 5. 重置 — 返回隔离交接

用户明确要求严格模式时，处理完当前段落就停止，并把以下交接返回给 Main：

> 当前段落已完成。后续隔离任务可以从[章节名]继续。

## 证据边界

不得生成占位或虚假的事实、引用、测量结果或数字。证据不足时，省略或标明不受支持的主张，并把准确的证据缺口返回给 Main。

## 中文写作规范

写作时遵守 `plain-chinese-writing` 技能中的相关规范。以下是要点提醒。

### 标点规则
- 破折号完全禁用
- 冒号限制使用（列举三项以上、引出定义时可用）
- 括号限制使用（缩写定义、编号可保留）
- 黑体限制使用（关键词定义、列表标签可保留）

### 语言风格
- 平直、准确、简洁
- 多用动词，少用抽象名词
- 语义完整，长短相间
- 注意中文特征：主题突出、意合衔接、主动优先
- 不为了规避标点规则而机械拆句
- 保留必要的限定、转折和承接信息
- 不机械使用"不在于……而在于……"转折模板
- 直接写具体困难、原因和后果，少用模板化转折
- "的"字不要连用三个以上

### 翻译腔和 AI 味
- 不堆叠名词，不滥用"进行""实现""基于"
- 不用英文语序，不用过多框式介词
- 不写空泛开头、泛化评价、套话结尾

## 反面模式

用户明确要求严格隔离模式时：

- 每个隔离循环只处理一段。
- 不把上一段的引用或来源当成本段事实证据。
- 来源中找不到引用标签时，删去对应论点或标明证据不足。
- 对当前段落做一次自检。
- 下一循环只加载该段所需上下文。

任何模式都不要把每个关键判断都写成"不是……而是……"或"不在于……而在于……"。

任何模式都只完成受派的 writer 局部方法。是否集成、要求独立 checker delivery 或开始另一项任务，由 Main 决定。
