# 中文严格写作状态机

严格状态机模式写中文。每段在独立的隔离上下文中生成：读取来源→写入→自检→保存→重置。每条引用、每个数字、每个事实论点都必须独立验证。章节主线、术语定义和用户已批准的写作风格可以继承，避免段落之间割裂。

## 适用场景

- 写实验结果、数据章节、或任何含引用和数值论点的内容
- 用户明确要求"严格模式"或"无幻觉"写作
- 多段落文档中段落间不允许上下文漂移

## 核心流程：每段一个独立循环

对每一段，独立处理，然后请用户继续。

### 1. 读取 — 只读当前需要的

只读当前这一段需要的源数据。事实证据只依赖当前段落的来源。可以读：
- 目标文档（扫描章节结构、术语定义和相邻段落的衔接）
- 当前段落的源文档、数据表或引用文献
- 章节的元数据/描述（如果有）

### 2. 写作 — 恰好一段

输出一段，结构如下：

```markdown
###### 标题 — 主题句（≤50字符）

正文长度服从语义完整和证据表达，含正确的 `[@引用]` 标记。
```

约束：
- 标题 ≤50 字符
- 正文语义完整，不为了变短而拆散论证
- 引用必须精确，从来源复制，不要改写引用标签
- 元数据记录在 `.pi/research/review_log.md`，不要在文档中用 HTML 注释

### 3. 自检 — 强制验证矩阵

| 检查项 | 通过/失败 |
|--------|-----------|
| 标题 ≤50 字符 | — |
| 正文语义完整，没有机械短句化 | — |
| 正文中每个 `[@...]` 引用在来源中存在 | — |
| 来源已记录在 review_log.md 并附带真实路径 | — |
| 没有超出源数据支持的论点 | — |
| 没有翻译腔表达 | — |
| 没有 AI 味表达 | — |
| 没有破折号 | — |
| 冒号、括号、黑体使用合理 | — |

任何一项失败，重写段落，不要继续。

### 4. 保存 — 插入文档

用精确的 edit 把新段落插入正确的章节标题下。

### 5. 重置 — 清除所有上下文

不要在当前会话中继续写下一段。输出：

> ✅ 段落已插入。请告诉用户"从[章节名]继续写作"。

## 中文写作规范

写作时遵守 `/skill:plain-chinese-writing` 的完整规范。以下是要点提醒。

### 标点规则
- 破折号完全禁用
- 冒号限制使用（列举三项以上、引出定义时可用）
- 括号限制使用（缩写定义、编号可保留）
- 黑体限制使用（关键词定义、列表标签可保留）

### 语言风格
- 平直、准确、简洁
- 多用动词，少用抽象名词
- 语义完整，长短相间
- 注意中文特征：主题突出、意合衔接、主动优先
- 不为了规避标点规则而机械拆句
- 保留必要的限定、转折和承接信息
- 不机械使用"不在于……而在于……"转折模板
- 直接写具体困难、原因和后果，少用模板化转折
- "的"字不要连用三个以上

### 翻译腔和 AI 味
- 不堆叠名词，不滥用"进行""实现""基于"
- 不用英文语序，不用过多框式介词
- 不写空泛开头、泛化评价、套话结尾

## 反面模式

- 绝不在一个会话中写两段
- 绝不把上一段的引用或来源参考当成本段事实证据
- 绝不猜测引用标签，来源中找不到就删除该论点
- 绝不跳过自检矩阵
- 绝不复用之前段落的读取上下文
- 绝不把每个关键判断都写成"不是……而是……"或"不在于……而在于……"
