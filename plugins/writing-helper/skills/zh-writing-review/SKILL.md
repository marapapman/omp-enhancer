# 中文写作审查

## 执行者边界

读取本 Skill 只为准备 writer assignment；仅凭这次读取不会让 Main 成为执行者。
本技能正文是受派写作子 Agent 的有界局部方法，在 Main
选定工作流和 Skill 后使用；本 Skill 不选择或调度 Agent，选择与调度仍由
Main 负责。单轮方法不能满足或替代独立 checker checkpoint，写作者的局部自检不能替代独立 checker。
Main 保留父级 TODO、finding disposition、集成、最终验证和面向用户
的交付权。

用户指定了文件或段落时，直接审查该目标。只有用户提供或明确引用
`.pi/research/checker_report.md`，或者当前工作流已经给出该报告时才使用它。
不要为了判断报告是否存在而主动探测 `.pi` 目录。

用户无需显式要求委派。目标长度只限制执行方法和 findings 数量，不构成
Main 直接执行的 fallback 理由。只读交付、统一的最终响应和协调开销也不构成
fallback 理由。受派 writer 返回完整的请求建议稿；选定工作流
保留 writer delivery、依赖后的 checker delivery 和父级集成 checkpoint。
只有当前许可限制确实阻止 dispatch 时，Main 才记录该限制，并只对受影响的
checkpoint 使用安全的直接 fallback。
`zh-writing-checkers` 是宽范围审查 Skill，不是 `zh-checker` Agent。局部润色
不需要加载这个宽范围 Skill；在工作流的软默认下，Main 可以从当前暴露的
Agents 中自主选择 `zh-checker` 做有界的语义漂移、逻辑和清晰度核查，本 Skill
不选择或调度该 Agent。Main 侧的 Skill 加载、writer 局部自检或宽范围
`zh-writing-checkers` Skill 加载，都不构成独立 checker 的执行或证据。

writer 子 Agent 始终只交付建议稿。按 assignment 返回完整建议文本；有界变更
使用 SEARCH/REPLACE 块或 unified diff 更清楚时，采用该格式。Main 保留权限决策
和实际文件修改。writer 不创建或持久化 review 产物。

## 受派 Writer 子 Agent 单轮方法

1. 只读取一次指定目标，并记录原文的语义锚点，包括限定、模态、范围、否定、方向、数字和单位、引用和标识符，以及 LaTeX 数学、交叉引用、命令和结构。
2. 按严重、重要、轻微整理问题。同一级别保留原顺序。
3. 用户明确要求修改或润色时，才在已有授权范围内提出表达、语法、结构、语气和格式修订，并保留事实和论点。仅审查任务只返回问题，不改写整篇正文。
4. 不擅自改变事实声明、核心论点、引用含义或任务范围。把这些项目列为作者决定。
5. 拟稿后只比对一次语义锚点。把任何漂移作为建议报告，不自动开启新的修复轮次。
6. 汇总已处理问题、未决问题和证据局限。

## 文本证据准确性

- 引号中的文字必须逐字来自成功读取的正文。最终响应前只核对一次每条引文和位置，不在正文中的内容必须删除或更正。
- 读取结果没有给出行号时，不得编造行号。概述或转述要明确标为转述，不能放进引号冒充原文。
- 区分“范围不清”和“相互矛盾”。只有主语与适用范围相同的两种表述确实不相容时，才称为矛盾。
- 建议改写是编辑建议，不是原文事实证据。不要把推测出来的具体原因写成原文已经表达的内容。

一轮最多处理 20 项。剩余问题只做汇总，不自动开启新的审查和修复循环。

只读审查任务在最终响应中返回有文本证据的问题；只有局部替换建议能帮助说明问题时才给出局部建议。用户没有明确要求重写时，不附整段或整篇改写稿。不创建 `.pi` 文件，也不为技能模板请求写权限。

成功读取已经到达目标文件末尾时，这一次读取足以完成有界审查。只有结果明确显示截断标记或请求范围不完整时，才换 selector 重读。除非缺少定义会导致对可见文字作出具体错误判断，否则不搜索宏定义或无关仓库上下文。

审查一个摘要或其他短文本时，受派 writer 子 Agent 最多报告五个实质问题。语义锚点清单只用于内部核对，不在最终响应中逐项复述；省略没有问题的严重级别和原文全文。完整目标已经足以支持问题时，立即收束这一子 Agent pass 并把 delivery 返回 Main。这只完成 writer 子 Agent 的有界审查；既不完成父级工作流，也不替代独立 checker delivery。

## 可选逐项模式

只有用户明确要求逐项审查，或确有一个会实质改变文本的选择时，才逐项询问接受、修改或跳过。用户已经要求润色或修改时，不需要为每个安全表达修改再次确认。

## 记录

当前工作流确实使用 `.pi/research/review_log.md` 时，返回已处理、待决定或跳过问题的建议记录。Main 决定是否在获授权的集成阶段持久化。不要只为满足本技能而创建工作流文件。

用户明确要求额外的宽范围审查时，Main 可以在 PLAN 阶段通过运行时正常的 Skill 机制声明并加载 `zh-writing-checkers`；受派 writer 子 Agent 不自行加载，也不把该 Skill 正文当作独立 checker 的执行证据。旧式斜杠技能写法只是文档，不是需要尝试的命令。

assignment 已包含并加载 `plain-chinese-writing` 时，写作遵守其相关规范；本 Skill 不触发额外加载。
